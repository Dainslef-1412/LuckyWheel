# 需求文档：单文件离线自定义转盘生成器

## 1. 项目概述
本应用是一个基于 Web 的工具，允许用户自定义转盘选项、权重及主题，并能将配置好的转盘导出为**独立的 HTML 文件**。导出的文件不依赖任何服务器或外部库，可在本地浏览器中直接运行、游玩及再次编辑。

## 2. 核心功能需求

### 2.1 配置模式 (Editor Mode)
* **标题自定义**：支持输入转盘的主题文字（如“今晚吃什么”）。
* **选项管理**：
    * 动态添加/删除选项。
    * **权重设置**：用户输入正整数作为权重（非百分比），系统自动根据总权重计算各扇区角度。
* **主题选择**：预设 3-5 组配色方案（如：活力、深邃、小清新）。
* **实时预览**：配置参数时，旁边的 SVG 转盘会根据权重实时调整扇区比例。

### 2.2 生成与导出 (Export Logic)
* **单文件封装**：点击“生成”后，应用将当前的配置数据（JSON 格式）注入预设的 HTML 模板字符串中。
* **本地保存**：利用 `Blob` 对象和 `URL.createObjectURL` 技术，触发浏览器下载一个 `.html` 文件。

### 2.3 游玩模式 (Play Mode)
* **转盘交互**：点击转盘或中心按钮开始旋转。
* **物理效果**：转盘需有由快到慢的减速动画过程（CSS `cubic-bezier` 或 JS 帧动画）。
* **结果展示**：
    * **中心显示**：转盘停止后，中心轴心区域显示中奖选项的文字。
    * **中奖判定**：旋转停止的角度必须精确对应权重计算出的扇区区间。
* **二次编辑**：页面提供“返回修改”按钮，点击后能切换回配置模式。

## 3. 技术规格

### 3.1 视觉表现 (Visuals)
* **图形渲染**：使用原生 **SVG** 绘制转盘。
    * 利用 $M (MoveTo)$, $L (LineTo)$, $A (Arc)$ 命令根据权重比例生成扇区路径。
* **适配性**：中心结果文字需具备简单的自适应逻辑（如根据文字长度缩放 `font-size`）。

### 3.2 数据结构 (Data Structure)
导出的单文件中应包含类似以下的配置对象：
```javascript
const CONFIG = {
    title: "我的转盘",
    theme: "forest",
    items: [
        { label: "选项A", weight: 1, color: "#2d5a27" },
        { label: "选项B", weight: 3, color: "#90be6d" }
    ]
};
```

### 3.3 算法逻辑
1.  **扇区角度计算**：
    * 总权重 $W = \sum w_i$。
    * 扇区 $i$ 的角度比例 $= \frac{w_i}{W} \times 360^\circ$。
2.  **命中算法**：
    * 随机生成 $0$ 到 $W$ 之间的数值 $r$。
    * 通过累加权重判定 $r$ 落在哪个区间，从而确定目标扇区。
3.  **旋转偏移**：
    * 最终旋转总度数 = $360 \times 圈数(如5圈) + (360 - 目标扇区中心角度)$。

## 4. 界面流程 (UI Flow)

1.  **首页 (未配置状态)**：显示空白表单和默认转盘。
2.  **配置中**：用户填写数据，转盘随之变化。
3.  **生成成功**：提示下载文件，并提供“预览游玩”按钮。
4.  **离线运行 (已下载的文件)**：
    * 默认展示“游玩界面”。
    * 点击“编辑” -> 切换至“配置界面” -> 重新修改 -> 点击“保存” -> 下载新版本。

---

## 5. 项目难点与注意事项
* **自包含性**：严禁引用任何外部 CDN（如 Google Fonts 或 jQuery），所有 CSS 和 JS 必须内联在生成的 HTML 字符串内。
* **权重精度**：需确保 SVG 扇区无缝拼接，且随机概率与权重比例严格一致。
* **浏览器兼容性**：导出的文件需在主流移动端和 PC 端浏览器（Chrome, Safari, Edge）中运行良好。
